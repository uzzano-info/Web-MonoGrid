# 🌐 MonoGrid 배포 및 기술 가이드 (Obsidian)

이 문서는 **MonoGrid** 프로젝트의 배포 과정과 라우팅 문제 해결 방법을 정리한 기술 가이드입니다.

---

## 1. 🚦 React 라우팅 문제와 해결 방법

### 🔍 문제의 현상 (What happened?)
웹사이트를 GitHub Pages에 배포했을 때, 메인 화면이 **하얀색(Blank Page)**으로 나오고 아무것도 뜨지 않는 문제가 발생했습니다. 브라우저 콘솔에는 `index-xxx.js` 파일을 찾을 수 없거나 경로가 잘못되었다는 오류가 가득했습니다.

### 🛠 해결 방법 (How to fix it?)
두 가지 설정이 필요합니다:

1.  **Vite 설정 (`vite.config.js`)**:
    ```javascript
    export default defineConfig({
      base: '/Web-MonoGrid/', // 프로젝트 저장소 이름
      // ...
    })
    ```
2.  **React Router 설정 (`App.jsx`)**:
    ```javascript
    <Router basename="/Web-MonoGrid/">
      <Routes>...</Routes>
    </Router>
    ```

### 💡 개념적 이해 (Easy Concept)
이 문제를 **'주소 체계의 오해'**로 이해하면 쉽습니다.

*   **일반적인 집 (로컬 환경)**: 프로젝트가 내 컴퓨터의 중심(`localhost:5173/`)에 살고 있습니다. 이때는 모든 방(페이지)의 문을 그냥 `"/"`라고 불러도 잘 찾아갑니다.
*   **아파트 안의 집 (GitHub Pages)**: GitHub Pages라는 큰 아파트 단지가 있고, 우리 프로젝트는 그 안의 **'Web-MonoGrid'**라는 호수(서브디렉토리)에 살고 있습니다.
    *   **문제 발생**: 프로젝트가 자기 주소를 여전히 `"/"` (아파트 로비)로 알고 있다면, 정작 자기 집(서브디렉토리) 안에 있는 가구(이미지, 코드)를 아파트 로비에 가서 찾으려고 합니다. 당연히 아무것도 없겠죠?
    *   **해결 방법**: `basename` 설정을 통해 프로젝트에게 "너는 아파트 로비가 아니라 **'Web-MonoGrid' 호수**에 살고 있어"라고 알려주는 것입니다. 그러면 프로젝트는 모든 물건을 자기 집 안에서 제대로 찾기 시작합니다.

## 🚀 2. 웹사이트 배포 과정 (Git & Deployment)

배포는 **"내 컴퓨터의 코드를 세상 사람들이 볼 수 있도록 서버에 올리는 과정"**입니다. 이번 프로젝트에서는 **GitHub Actions**를 사용해 자동 배포를 구축했습니다.

### 📝 Git 커밋 방법 (저장하는 방법)
코드에 변화를 준 후, 이를 기록하고 서버로 보내는 기본적인 3단계입니다.

1.  **`git add .` (준비물 챙기기)**:
    *   수정된 파일들을 "이제 곧 저장할 거야"라고 목록에 올리는 과정입니다. (장바구니에 담기)
2.  **`git commit -m "설명"` (도장 찍기)**:
    *   지금의 변화를 확정 짓는 기록입니다. 나중에 문제가 생겼을 때 이 기록을 보고 되돌아갈 수 있습니다. (결제 완료)
3.  **`git push origin main` (트럭 출발)**:
    *   내 컴퓨터에만 있던 기록들을 GitHub라는 멀리 있는 창고로 보내는 과정입니다.

### 🏗 배포 자동화 과정 (Deployment Flow)
우리가 설정한 **GitHub Actions** 덕분에 이제는 'Push'만 하면 모든 것이 자동으로 이루어집니다.

1.  **Push (방화쇠)**: 내가 코드를 GitHub에 올립니다.
2.  **Action (공장 가동)**: GitHub 서버가 자동으로 깨어나서 가상 컴퓨터를 한 대 만듭니다.
3.  **Build (제품 조립)**: 그 가상 컴퓨터가 `npm install`과 `npm run build`를 실행하여 소스코드를 압축하고, 브라우저가 읽기 쉬운 형태(`dist` 폴더)로 변형합니다.
4.  **Deploy (전시)**: 조립이 끝난 결과물을 GitHub Pages 서버에 올립니다.
5.  **Live (완료)**: 이제 인터넷 주소를 통해 누구나 내 사이트를 볼 수 있게 됩니다.

---

> [!TIP]
> 이제 코드를 수정하고 **Push**만 하면, GitHub가 알아서 빌드하고 배포해 줍니다. 배포 상태는 GitHub 저장소의 **'Actions'** 탭에서 실시간으로 확인할 수 있습니다.
